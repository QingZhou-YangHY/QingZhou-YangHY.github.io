知识点总结

重点概念

1. 数据结构

定义: 数据结构是相互之间存在一种或多种特定关系的数据元素的集合。

分类:

  • 逻辑结构: 线性结构、树形结构、图结构
  • 物理结构: 顺序存储、链式存储

2. 算法复杂度

时间复杂度: 算法执行时间与输入规模的关系 空间复杂度: 算法运行时占用存储空间与输入规模的关系

重要公式

常见时间复杂度

$$O(1) < O(\log n) < O(n) < O(n \log n) < O(n^2) < O(2^n) < O(n!)$$

主定理

对于递归关系式 $T(n) = aT(n/b) + f(n)$：

$$T(n) = \begin{cases} O(n^{\log_b a}) & \text{if } f(n) = O(n^{\log_b a - \epsilon}) \\ O(n^{\log_b a} \log n) & \text{if } f(n) = \Theta(n^{\log_b a}) \\ O(f(n)) & \text{if } f(n) = \Omega(n^{\log_b a + \epsilon}) \end{cases}$$

def binary_search(arr, target):
    """
    二分查找算法实现
    时间复杂度: O(log n)
    空间复杂度: O(1)
    """
    left, right = 0, len(arr) - 1
    
    while left <= right:
        mid = (left + right) // 2
        
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    
    return -1

# 测试用例
arr = [1, 3, 5, 7, 9, 11, 13, 15]
result = binary_search(arr, 7)
print(f"元素7的位置: {result}")  # 输出: 3

def quick_sort(arr, low=0, high=None):
    """
    快速排序算法实现
    平均时间复杂度: O(n log n)
    最坏时间复杂度: O(n²)
    """
    if high is None:
        high = len(arr) - 1
        
    if low < high:
        # 分割数组，返回基准元素位置
        pivot_index = partition(arr, low, high)
        
        # 递归排序基准元素左右两部分
        quick_sort(arr, low, pivot_index - 1)
        quick_sort(arr, pivot_index + 1, high)

def partition(arr, low, high):
    """分割函数"""
    pivot = arr[high]  # 选择最后一个元素作为基准
    i = low - 1  # 小于基准元素的索引
    
    for j in range(low, high):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]
    
    arr[i + 1], arr[high] = arr[high], arr[i + 1]
    return i + 1

def example(n):
    sum = 0
    for i in range(n):
        for j in range(i):
            sum += i * j
    return sum

解题思路: 分析嵌套循环的执行次数

解答过程: 外层循环执行n次，内层循环分别执行0, 1, 2, …, n-1次 总执行次数 = 0 + 1 + 2 + … + (n-1) = n(n-1)/2 因此时间复杂度为 O(n²)

知识点: 嵌套循环复杂度分析

难点分析

  3. 空间复杂度: 区分递归调用栈和额外存储空间
